# Документация MDM

Актуальность: март 2026.

## Состав
- `concept.md` — архитектурная концепция и целевая модель.
- `requirements.md` — функциональные/нефункциональные требования и текущий статус реализации.
- `rest-api.md` — человекочитаемое описание REST API.
- `openapi.yaml` — контракт API в формате OpenAPI 3.0.3.
- `backend-e2e.md` — статус последнего сквозного e2e тестирования backend API и зафиксированные аномалии.
- `schema.sql` — актуальная DDL-схема БД.
- `er.mmd` — ER-диаграмма.
- `ui.md` — требования к подпроекту `ui` (Vue), UX/UI и покрытию API.

## Важные договоренности
- MDM работает за gateway, интегрированным с Keycloak.
- Аутентификация/авторизация в MDM строится на заголовках:
  - `X-User-Id`
  - `X-User-Role`
- `X-Request-Id` опционален: при отсутствии/невалидном UUID сервер генерирует новый.
- В MDM нет локальных таблиц `users/roles`.

## Миграции
- Для текущего этапа используется единая SQL-миграция:
  - `core/internal/migrations/0001_init.sql`
- Каталог миграций в коде: `core/internal/migrations`.
